# External AI Tools Protocol

The UPHSL GDC engineering team leverages various AI protocols (like GitHub Copilot or Claude Code skills) inside repository automation workflows.

## Using AI the right way
1. **Component Generation:** You are highly encouraged to use LLMs to scaffold out tedious React standard boilerplates or repetitive Tailwind class combinations.
2. **Schema Logic Check:** Never allow an AI parameter map to fully dictate Supabase Database Table Migrations blindly. Always manually trace relationships (Foreign Keys & Cascade deletes) yourself to prevent irreversible table drop schemas.
3. **Internal Tools (.claude/skills/):** We maintain prompt-shells matching ADR generations automatically. If you construct an ADR using local scripts, double-check that the AI didn't hallucinate non-existent Vercel environments or random databases. Ensure outputs align with `docs/reference/SYSTEM-ARCHITECTURE.md`.
