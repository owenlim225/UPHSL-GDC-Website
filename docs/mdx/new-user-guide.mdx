# New Developer Onboarding Guide

Welcome to the development team at the UPHSL Game Developers' Club!

## Day 1: Access Check

To effectively execute logic on the platform you will need:
1. **GitHub Access:** Ensure your username is mapped to the `uphsl-gdc` repository organizations.
2. **Supabase Access:** Ask an active Officer to invite your school email into the Supabase project workspace. You need this to verify SQL migrations and read Row-Level Security parameters.
3. **Vercel Access (Optional):** Only leads strictly need Vercel access for log-tailing; standard devs can manage deployments seamlessly via GitHub PR previews natively.

## Day 2: Environment Bootstrapping

Review the core [Getting Started Guide](./getting-started.mdx) to get `npm run dev` firing.
Open the project locally and begin familiarizing yourself with the `src/app` architecture layout mapping out public vs grouped `(dashboard)` nested routes.

## Day 3: Pick up your first issue!

Check out GitHub's issue tracker labeled with `good-first-issue`, or message the Lead UI/UX Officer in Discord. Often the best starting points are converting new Figma layouts strictly into responsive `Tailwind` static React components.
