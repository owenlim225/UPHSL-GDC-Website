# GDC Platform Planning Framework

As opposed to rigid corporate planning templates, the UPHSL Game Developers' Club platform relies heavily on Agile iteration. 

## 1. Feature Propositions
Any Club Member can propose platform improvements directly on the GitHub Repository's Issue Tracker or the Discord `-web-dev` channel.

## 2. Architectural Approval
If the feature strictly impacts database schemas or external networking (e.g. "We want itch.io integration!"), an active Technical Officer must review and write an **Architecture Decision Record (ADR)** inside `/docs/adr/`. 

## 3. Scope Definition
- Scope must be cleanly split between **Public Site** (viewed by all students) and **Dashboard Platform** (strictly authenticated context logic). The public site must prioritize loading speed above component reusability.

## 4. UI / UX Design Process
Before committing React code, interfaces should be broadly modeled in our Figma workspace, strictly adhering to the exported token boundaries in `design-system.json`. Tailwind classes (`p-4 bg-background`) should mirror the Figma properties directly.

## 5. Development Cycle
1. Define the SQL schema and Row Level Security logic first inside Supabase.
2. Regenerate TypeScript typings bindings.
3. Execute Next.js routing maps in `src/app`.
4. Build localized React components in `src/components`.
5. Connect Server Actions for client forms to database updates.
