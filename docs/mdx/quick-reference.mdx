# Quick Reference Cheat Sheet

Daily reference commands and scripts for the UPHSL GDC platform developers.

## Local Server Commands

- `npm run dev` - Spins up the Next.js development server locally.
- `npm run build` - Validates type-checks and executes a production build compile.
- `npm run lint` - Runs ESLint against strict rulesets over `src/`.

## Supabase CLI Essentials

Ensure you have the Supabase CLI installed if creating new migrations.
- `supabase init` - Re-initialize local configs (rarely needed).
- `supabase start` - Boots up a local, isolated Docker representation of Supabase to test destructive schema changes without touching the live Production database.
- `supabase migration new add_events_table` - Drafts a new SQL file you can populate with raw DDL before syncing it upward.
- `supabase gen types typescript --project-id XYZ > types/supabase.ts` - Refreshes TypeScript types matching current PostgreSQL schema parameters!

## Environment Flags

- `.env.local`: Add development keys. **NEVER** commit this file.
- `.env.example`: Safe representations of existing key mapping arrays.

## Design Tokens & Styling

- **`design-system.json`**: Primary source of truth for branding.
- **Note:** Modifying JSON tokens may require a **restart** of the `npm run dev` server for changes to re-inject into the Tailwind CSS variable layer.

## Git Workflow
- Create branches off `main` -> `feature/event-calendar`. 
- Commit with conventional tags -> `feat: adds public calendar route`.
- Open a PR to `main` to trigger Vercel Preview Deployments.
- Review [ADR-001: Initial Tech Stack](../adr/adr-001-initial-tech-stack.md) for architectural rationale.
