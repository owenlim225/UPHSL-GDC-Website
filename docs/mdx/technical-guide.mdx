# Technical Implementation Guide

This guide details the core technical implementation standards for the UPHSL Game Developers' Club platform.

## 1. Frontend Standards (Next.js & React)

- **App Router:** We exclusively use the Next.js App Router (`src/app`). Do not use the legacy `pages` directory.
- **Server Components:** By default, all components should be Server Components (`React Server Components`). Only use `'use client'` at the lowest possible branches of the UI tree when states, hooks, or event listeners are fundamentally required.
- **Data Fetching:** Do not use `fetch()` wrappers globally on the client side unless you have to. We prefer Server Actions to securely hit Supabase logic directly from the Next.js node layer.

## 2. Authentication Flow (Supabase)

- **Middleware:** Auth is protected via Next.js Edge Middleware. Routes under `/dashboard` automatically enforce active Supabase Session checks.
- **Session Tokens:** JWTs are bound automatically in browser local-storage/cookies via `@supabase/ssr`.

## 3. Database & SQL Policy

All interactions to the Database MUST respect Row Level Security (RLS). 
- **Read Logic:** If a student loads their personal `games` array, Supabase RLS policies check incoming authenticated UUIDs to restrict row visibility.
- **Admin Overrides:** Features residing under `/dashboard/admin/*` utilize the `SUPABASE_SERVICE_ROLE_KEY` internally to bypass RLS entirely when Officers need global views. **Never** expose this key to the client.

## 4. UI Library

We use standard Tailwind CSS.
Do not introduce alternative CSS modules or frameworks like ChakraUI or MaterialUI. If complex widgets are required, extend from `shadcn/ui` copying the raw React components directly into `components/ui/`.
